# LocalText2Voice v0.4.1

Chatterbox installation hotfix for Windows x64.

## Fixes

- Chatterbox now uses **Auto** as the default compute device instead of strict
  CUDA.
- Existing settings from v0.4.0 that saved Chatterbox as `cuda` are migrated to
  `auto`.
- If a CUDA GPU is not visible, the Chatterbox installer retries with Auto/CPU
  instead of failing after the runtime download.
- Chatterbox generation also retries with Auto/CPU when an older downloaded
  runtime reports that CUDA is unavailable.
- The Chatterbox runtime CLI now falls back to CPU when `--device cuda` is
  requested on a PC without CUDA.

## Install

1. Download `LocalText2Voice-v0.4.1-windows-x64.zip`.
2. Extract the ZIP.
3. Run `LocalText2Voice.exe`.
4. In **Settings > Voice Generation Engine > Chatterbox**, leave **Compute
   device** on **Auto (recommended)** unless you specifically want to force a
   device.
